<h3 id="GfCCk">**<font style="color:rgb(51, 51, 51);">(二) </font>**<font style="color:rgb(79, 79, 79);">同步传输、异步传输</font></h3>
---

**<font style="color:rgb(77, 77, 77);">1. 同步传输</font>**

<font style="color:rgb(77, 77, 77);">同步传输：数据块以稳定的比特流形式传输，字节之间没有间隔，接收端在每个比特的中间时刻进行检测，以判别接收到的是比特0或是比特1。  
</font>![](https://cdn.nlark.com/yuque/0/2025/png/53755062/1742957266883-d685f532-7e12-4e29-8db4-431251a19d80.png)

> <font style="color:#000000;">由于不用设备的时钟频率存在差异，传输大量数据时，所产生判别时刻的累计误差会导致对比特信号的判别错位，需要采取方式使双方时钟同步。</font>
>

<font style="color:rgb(77, 77, 77);">实现收发双方时钟同步的方法</font>

+ `<font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">外同步</font>`<font style="color:rgba(0, 0, 0, 0.75);">：在收发双方之间添加一条单独的时钟信号线</font>
+ `<font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">内同步</font>`<font style="color:rgba(0, 0, 0, 0.75);">：发送端将时钟同步信号编码到发送数据中一起传输（例如传统以太网所采用的曼切斯特编码）</font>



**<font style="color:rgb(77, 77, 77);">2. 异步传输</font>**

<font style="color:rgb(77, 77, 77);">异步传输：以字节为独立的传输单位，字节之间的时间间隔不是固定的。接收端仅在每个字节的起始处对字节内的比特实现同步。为此通常在每个字节的前后分别加上起始位和结束位。  
</font>![](https://cdn.nlark.com/yuque/0/2025/png/53755062/1742567440160-5d670542-e64b-47b9-aed9-a82d82d512b8.png)

+ <font style="color:rgba(0, 0, 0, 0.75);">字节之间异步（字节之间的时间间隔不固定）</font>
+ <font style="color:rgba(0, 0, 0, 0.75);">字节中的每个比特仍然要同步（各比特的持续时间相同）</font>

<font style="color:rgb(77, 77, 77);">在计算机网络中，计算机需要处理和传输用户的文字、图片、音频和视频，它们可以统称为 </font>`<font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">消息</font>`<font style="color:rgb(77, 77, 77);">。</font>`<font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">数据是运送消息的实体</font>`<font style="color:rgb(77, 77, 77);">。计算机中的网卡将比特0和1变换成相应的 </font>`<font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">电信号</font>`<font style="color:rgb(77, 77, 77);"> 发送到网线，也就是说 </font>`<font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">信号是数据的电磁表现</font>`<font style="color:rgb(77, 77, 77);">。</font>

<font style="color:rgb(77, 77, 77);">由信源发出的原始电信号称为 </font>`<font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">基带信号</font>`<font style="color:rgb(77, 77, 77);"> ，基带信号分为 </font>`<font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">数字基带信号</font>`<font style="color:rgb(77, 77, 77);">、</font>`<font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">模拟基带信号</font>`<font style="color:rgb(77, 77, 77);"> 两类。</font>

<font style="color:rgb(77, 77, 77);">信号需要在</font>`<font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">信道</font>`<font style="color:rgb(77, 77, 77);"> 中传输，信道可分为 </font>`<font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">数字信道</font>`<font style="color:rgb(77, 77, 77);">和 </font>`<font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">模拟信道</font>`<font style="color:rgb(77, 77, 77);"> 两种。</font>

<h3 id="IhnlD">**<font style="color:rgb(51, 51, 51);">(二) </font>**<font style="color:rgb(79, 79, 79);">常用编码</font></h3>
---

<h4 id="IQWg1">**<font style="color:rgb(77, 77, 77);">不归零编码（NRZ）：存在同步问题</font>**</h4>
<font style="color:rgb(77, 77, 77);">所谓不归零，就是指在整个码元时间内，电平不会出现零电平</font>![](https://cdn.nlark.com/yuque/0/2025/png/53755062/1742568460633-e257d933-2eb0-4043-9ae9-5f6d259138fd.png)

+ `<font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">需要额外一根传输线来传输时钟信号</font>`<font style="color:rgba(0, 0, 0, 0.75);">，接收方按时钟信号的节拍来逐个接收码元</font>
+ <font style="color:rgba(0, 0, 0, 0.75);">由于不归零编码不能葱波形本身中提取信号时钟频率（这叫做没有自同步能力）</font>

<font style="color:rgba(0, 0, 0, 0.75);"></font>

<h4 id="kVciA"><font style="color:rgba(0, 0, 0, 0.75);">归零编码（RZ）：自同步，编码效率低</font></h4>
<font style="color:rgba(0, 0, 0, 0.75);">每个码元传输结束后信号都要 “归零”，所以接收方只要在信号归零后进行采样即可，不需要单独的时钟信号。</font>

+ <font style="color:rgba(0, 0, 0, 0.75);">实际上，归零编码相当于将时钟信号用 “归零” 方式编码在了数据之内，这种称为 “自同步” 信号</font>
+ <font style="color:rgba(0, 0, 0, 0.75);">但是，归零编码中大部分的数据带宽都用来传输 “归零” 而浪费</font>

![](https://cdn.nlark.com/yuque/0/2025/png/53755062/1742568576306-6fc92aab-90de-43e3-acea-1125a5f472fe.png)

<h4 id="UtbGl">**<font style="color:rgb(77, 77, 77);">曼彻斯特编码</font>**</h4>
<font style="color:rgb(77, 77, 77);">在每个码元时间的中间时刻，信号都会发生跳变。例如，负（下）跳变表示比特 1，正（上）跳变表示比特 0，根据惯性思维相反来记。</font>

+ <font style="color:rgba(0, 0, 0, 0.75);">码元中间时刻的跳变既表示时钟，又表示数据</font>
+ <font style="color:rgba(0, 0, 0, 0.75);">传统以太网就使用的曼切斯特编码</font>

![](https://cdn.nlark.com/yuque/0/2025/png/53755062/1742958771489-45d1856d-5552-46b9-b19e-363b1f21c1ad.png)



<h4 id="i6Stq">**<font style="color:rgb(77, 77, 77);">差分曼彻斯特编码</font>**</h4>
+ <font style="color:rgba(0, 0, 0, 0.75);">每个码元中间都有跳变仅表示时钟</font>
+ <font style="color:rgba(0, 0, 0, 0.75);">第一个码元遵循曼彻斯特编码</font>
+ <font style="color:rgba(0, 0, 0, 0.75);">码元开始处发生跳变表示 0，不跳变表示 1</font>

![](https://cdn.nlark.com/yuque/0/2025/png/53755062/1742958979496-6484aa19-b28f-46f3-85fd-fff437cbf0a6.png)

:::color5
**优点**

**抗干扰能力更强**：由于数据通过跳变的有无表示，而非跳变方向，因此对信号畸变的容忍度更高。

**同步更可靠**：每个比特位起始的跳变确保了时钟同步，减少了长连续相同数据导致的同步丢失问题。

在传输大量连续1或连续0的情况下，差分曼彻斯特编码信号比曼彻斯特编码信号的变化少，更适合较高的传输速率。

在噪声干扰环境下，检测有无跳变比检测跳变方向更不容易出错，因此差分曼彻斯特编码信号比曼彻斯特编码信号更易于检测。

在传输介质接线错误导致高低电平翻转的情况下，差分曼彻斯特编码仍然有效。

:::

<h3 id="BroIl">**<font style="color:rgb(51, 51, 51);">(三) </font>**<font style="color:rgb(79, 79, 79);">基本调制方法</font></h3>
---

基带信号往往包含较多的低频成分，甚至有直流成分，而许多的信道并不能传输这种低频分量或直流分量，因此必须对基带信号进行<font style="color:#DF2A3F;">调制</font>。

![](https://cdn.nlark.com/yuque/0/2025/png/53755062/1742959071904-14e7bf65-be17-427e-985e-0cae7122db09.png)	

+  基本调制属于二元制，也就是只能调制出两种基本波形，每种波形表示1比特的信息量
+ 基本调制包括 调幅AM、调频FM、调相PM。调幅信号有两种不同振幅的基本波形构成；调频信号由两种不同频率的基本波形构成；调相信号由两种不同初相位的基本波形构成
+ 如果要使一个码元包含多个比特信息，可以采用混合调制的方法



<h3 id="TmXBu">**<font style="color:rgb(51, 51, 51);">(四) </font>**<font style="color:rgb(79, 79, 79);">混合调制</font></h3>
---

:::color5
<font style="color:rgb(85, 86, 102);background-color:rgb(238, 240, 244);">混合调制，属于多元制，也就是可以调制出多种基本波形</font>

:::

因为频率和相位是相关的，即频率是相位随时间的变化率，所以一次只能调制频率和相位中的一个。通常情况下，相位和振幅可以结合起来一起调制，称为<font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">正交振幅调制 QAM</font>。



举例：正交振幅调制 QAM-16

+ 可以调制出 12 种相位
+ 每种相位有 1 或 2 种振幅可选
+ 因此共可以调制出 16 种码元（波形），每种码元可以对应表示 4 个比特

![](https://cdn.nlark.com/yuque/0/2025/png/53755062/1742569220755-fbb90c5a-8f80-4fb0-a1f6-c0a5952ff280.png)

+ <font style="color:rgba(0, 0, 0, 0.75);">码元与 4 个比特的对应关系不能随便定义，应采用格雷码。任意两个相邻码元之间只有 1 个比特不同

<h3 id="pnUjB">**<font style="color:rgb(51, 51, 51);">(二) </font>**<font style="color:rgb(79, 79, 79);">奈氏准则、香农公式</font></h3>
---

<font style="color:rgb(77, 77, 77);">奈奎斯特（Nyquist）定理又称 奈氏准则</font>![](https://cdn.nlark.com/yuque/0/2025/png/53755062/1742569295101-8ff3b419-e224-48c9-a72c-8044e39d8c48.png)![](https://cdn.nlark.com/yuque/0/2025/png/53755062/1746532704285-098ae0de-d8b1-4453-8039-b335a6d4a11b.png)

+ 码元传输速率 又称为 波特率、调制速率、波形速率 或 符号速率。它与比特比特率的关系：
    - 当1个码元只携带 1 比特的信息量时，则波特率（码元/秒）与比特率（比特/秒）在数值上是相等的
    - 当1个码元携带 n 比特的信息量时，则波特率转换成比特率时，<font style="background-color:#FBDE28;">数值要乘以 n</font>
+ 要提高信息传输速率（比特率)，就必须设法使每<font style="color:#DF2A3F;">一个码元能携带更多个比特的信息量</font>。这需要采用多元制，例如 QAM-16
+ 实际信道所能传输的码元速率，要明显低于奈氏准则给出的上限值。奈氏准则是理想条件下推导的，没有考虑传输距离、噪声干扰等因素

![](https://cdn.nlark.com/yuque/0/2025/png/53755062/1742569406284-dc0a7ac7-52f8-47fe-abe9-ce7a125d1638.png)

+ <font style="color:rgba(0, 0, 0, 0.75);">信道带宽或信道中信噪比越大，信道的极限传输速率越高</font>
+ <font style="color:rgba(0, 0, 0, 0.75);">在实际信道上能够达到的信息传输速率要比该公式的极限传输速率低不少。这是因为在实际信道中，信号还要受到其他一些损伤，如各种脉冲干扰、信号在传输中的衰减和失真等，这些因素在香农公式中并未考虑</font>

  
 ![](https://cdn.nlark.com/yuque/0/2025/png/53755062/1742569441192-1b8eacf7-331f-48bb-9485-5273ad8bf0b4.png)

+ <font style="color:rgba(0, 0, 0, 0.75);">在信道带宽一定的情况下，根据奈氏准则和香农公式，要想</font><font style="color:rgba(0, 0, 0, 0.75);"> </font>`<font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">提高信息的传输速率</font>`<font style="color:rgba(0, 0, 0, 0.75);"> </font><font style="color:rgba(0, 0, 0, 0.75);">就必须采用</font><font style="color:rgba(0, 0, 0, 0.75);"> </font>`<font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">多元制</font>`<font style="color:rgba(0, 0, 0, 0.75);">（更好的调制方法）和努力</font><font style="color:rgba(0, 0, 0, 0.75);"> </font>`<font style="color:rgb(199, 37, 78);background-color:rgb(249, 242, 244);">提高信道中的信噪比</font>`
+ <font style="color:rgba(0, 0, 0, 0.75);">自从香农公式发表后，各种新的信号处理和调制方法就不断出现，其目的都是为了尽可能地接近香农公式给出的传输速率极限</font>

<h3 id="I3A49">**<font style="color:rgb(51, 51, 51);">(一) </font>**频分复用（FDM, Frequency Division Multiplexing）</h3>
:::color5
频分复用是最基本的频率利用方式之一。它将整个带宽分为多份，用户在分配到一定的频带后，在通信过程中自始至终都占用这个频带。所有用户在同样的时间占用不同的带宽（即频带）资源。

:::

![](https://cdn.nlark.com/yuque/0/2025/png/53755062/1742643436695-fa806bb2-68cd-4506-860d-740f40cf0fd5.png)



<h3 id="obR1I">**<font style="color:rgb(51, 51, 51);">(二) </font>**时分复用 TDM(Time Division Multiplexing)</h3>
:::color5
+ 时分复用则是将时间划分为一段段等长的时分复用帧（TDM 帧），每一路信号在每一个 TDM 帧中占用固定序号的时隙（同步时分复用）；
+ 每一路信号所占用的时隙是周期性地出现（其周期就是 TDM 帧的长度），TDM 信号也称为等时信号；
+ 时分复用的所有用户是在不同的时间占用同样的频带宽度。

:::

![](https://cdn.nlark.com/yuque/0/2025/png/53755062/1742643559764-b6065e0b-645c-45b3-a6c6-20098bdd52db.png)![](https://cdn.nlark.com/yuque/0/2025/png/53755062/1742644481590-6438edda-42ce-42f3-927a-69246c6094ab.png)![](https://cdn.nlark.com/yuque/0/2025/png/53755062/1742713286909-159a8d4d-bc6e-4fb2-8436-757478ce9234.png)



:::color3
使用FDM或TDM的复用技术，可以让多个用户(可以处在不同地点)共享信道资源。

频分多址接入(Frequency Division Multiple Access)：<font style="background-color:#FBDE28;">频分多址</font> ——允许多个用户同时共享一个通信信道，通过将可用频谱划分为不同的频带（子信道），每个用户分配一个独立的频带进行通信。用户在各自的频带上传输信号，彼此之间不会干扰。

时分多址接入TDMA (Time Division Multiple Access)：<font style="background-color:#FBDE28;">时分多址</font> ——允许多个用户共享一个通信信道，通过将时间划分为连续的时隙，每个用户在特定的时隙内传输数据。

:::
